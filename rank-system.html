Rank System — design

XP-driven progression з 30 рангами, 6 tier groups, hybrid naming, season reset кожні 4 тижні. Замикає onboarding (gate-via-rank), retention (рости daily) і соц. proof (rank скрізь видно).

Архітектурний коміт. 30 рангів у 6 tier groups (Newbie→Mythic), display "Rank 7 · Trader II". XP від участі (10) + position multiplier (#1 = +100, top-30% = +5). Bear contests дають ×1.5 множник (incentivize ваш диференціатор). Контент gated by rank — Rank 1 має тільки Welcome Match, кожен наступний відкриває щось. Season = 4 тижні, в кінці soft reset: drop 5 ranks (capped at Rank 5 minimum) — ніхто не страждає, але є fresh grind. Career-highest зберігається як trophy.

XP curve (cumulative + delta per rank)

Симулятор: який rank досягається за 4-тижневий сезон?

Casual (1 contest/day, ~15 XP avg) → Rank 5 (Newbie V). Engaged (3/day, ~30 XP avg) → Rank 14 (Degen IV). Hardcore (5/day, ~80 XP avg, frequent top-5) → Rank 27 (Mythic II). Касуальний юзер встигає один раз "відчути progression" і отримати Bear unlock у перший сезон — ідеальна voronka.

XP per contest — formula

xp = (10 + position_bonus) × contest_multiplier
Component XP Logic
Participation +10 always, finalized only
#1 (winner) +100 position bonus
#2 +60
#3 +40
#4-5 +25
#6-10 +15
#11 → top 30% +5 still in prizes
Below top 30% +0 just participation

Contest multipliers

multiplier = base × bear_bonus × stake_bonus
Modifier × Why
Bull contest (default) ×1.0 baseline
Bear contest ×1.5 incentivize differentiator
Quick Match (low stakes) ×1.0 baseline
Degen-tier contest ×1.25 reward stake
Whale Vault ×1.5 premium
Legend League ×2.0 endgame

Multipliers множаться сумарно. Top-3 у Bear Whale Vault: (10 + 40) × 1.5 × 1.5 = 112.5 XP (округлено до 113).

Curve formula

xp_required(N) ≈ 30 × 1.22^(N-2)

Геометричний ріст. Деякі ranks вручну скоригованo для psychologically nice numbers (50 → 80 → 150 → 250). Базова умова: каждый next rank вимагає на ~22% більше XP.

Якщо хочеш softer curve (швидший Mythic) — змінюй b=1.18. Steeper grind (Mythic як end-game) — b=1.27.

Season mechanic

Тривалість. 4 тижні (28 днів). Прив'язка до календарних місяців: Season 1 = April 2026, Season 2 = May 2026.

Що скидається. users.xp_season → 0. users.current_rank → drop 5 (capped мінімум на Rank 5, щоб ніхто не повертався в onboarding).

Що зберігається. users.xp_total (career), users.career_highest_rank, всі trophy-badges за минулі сезони ("Reached Rank 24 in Season 1").

Що нагороджується. Top-100 season leaderboard (за xp_season) → bonus prize pool (Stars). Top-10 → permanent "Season Champion" badge з номером сезону.

Чому soft reset. Hard reset (всі → Rank 1) ламає content gating — veteran знову не може купити cup. Soft drop робить fresh grind, але baseline access зберігається. Це Hearthstone-pattern, перевірений.

Що відкривається на кожному рангу. Це тіло onboarding-funnel і long-term retention loop.

Як це працює як onboarding driver

Юзер новий → бачить лобі з одним unlocked-контестом + 4-5 locked teasers нижче з "🔒 Unlocks at Rank 2-5". Це не frustrating — це aspirational. Юзер чітко бачить shorted path: "грай → ранк → відкрилось". Перший контест дає 110 XP (winner) або 35 XP (top-5) → одразу Rank 2. Іnstant gratification + immediate next goal. Тaper з кожним рангом до Mythic, де grind стає 1+ тиждень на рівень.

Lobby — rank chipalways visible

Mini-pill зверху-праворуч: (7) Trader II · 240/350 XP. Чорний bg + accent-red rank circle = anchor для ока. Tap → відкриває окремий /me/rank screen з повною ladder. Завжди видно — і в Lobby, і у всіх скрінах де є Header (Live, Result).

Lobby — next-rank teaser banneronboarding

Жовтий paper note, як streak widget у профайлі. Точна headline формула: "Reach Rank N · TierName to unlock UnlockName". Прогрес-bar показує наскільки близько. Зникає коли юзер на Mythic V (немає більше unlock'ів) — тоді замінюється на "Defend your Mythic crown — N XP за сезон".

Lobby — locked contest cardsonboarding

Locked contests показуємо БЕЗ ховання. opacity: 0.85 + dim background + чорна "🔒 RANK 8" plate. Текст контесту dimmed. Сортування: unlocked спочатку, потім locked — у порядку зростання rank requirement. Це aspirational, не frustrating — юзер бачить що його чекає.

Result — XP breakdownretention

Окремий блок з code-bg background. Кожна XP-строка фейдиться по черзі при відкритті (stagger 100ms): Participation → Position bonus → Multiplier bonus → Total. Total великим accent кольором. Bonus modifiers (Bear ×1.5) кольоровані в FTL purple щоб підкреслити "ти вибрав Bear, ти молодець". Прогрес-bar показує both: де був до (full bg) + скільки додалось (накладений yellow segment) — visceral feedback.

Result — RANK UP overlayretention killer moment

Full-bleed accent gradient. Великі цифри ("RANK 8" — 48px). Триггериться ТІЛЬКИ якщо rank changed. Trigger order: спочатку показуємо звичайний result + XP breakdown, потім bar заповнюється до 100%, потім повноекранне overlay. Має tactile feedback через TG SDK Haptic notification('success'). CTA Customize now → веде юзера ВІДРАЗУ на новий unlock — не лиши момент висіти. Якщо unlock = новий контест → веде у lobby з підсвіченим.

Profile — rank sectionprofile anchor

Розташовується ПЕРЕД stats grid (вище секції 4). Tier icon великий (48px) — найбільший visual element на profile. Trophy line знизу — "Career high: Rank 12 · Degen II" — створює long-term aspirational. Якщо у юзера є season trophies (закінчені сезони) — додаємо рядок з emoji-стрічкою: 🥇🥉🥈... Tap на тier icon → opens full ladder screen.

Leaderboards — rank поряд з usernamesocial proof

На /rankings кожен юзер має mini rank-badge біля імені: (12) Andriy. Whale+ tier — gold-accent username (cosmetic unlock at Rank 20). Створює "wow ці люди серйозні" effect. Опційно: filter "Show only Whale+" або "My league" (current rank ±2).

Що ВАЖЛИВО для дизайнера

Rank chip і Profile rank section мають бути ВІЗУАЛЬНО CONNECTED. Той самий tier icon, той самий progress bar style. Юзер натискає chip → бачить розгорнуту версію того ж самого. Continuity → mental model. Tier icons можуть еволюціонувати від простих квадратів (Newbie) до складних геральдичних форм (Mythic) — це додаткова тиха progression visualization.

3. Backend — wire to contests.finalize

// contests.finalize.ts — extension async finalize(contestId) { // ... existing finalization (compute final_score, distribute prizes) for (const entry of realEntries) { const award = awardXp({ position: entry.position, totalRealUsers: realEntries.length, contestType: contest.type, contestMultiplier: contest.xpMultiplier, }); await db.transaction(async (tx) => { await tx.insert(xpEvents).values({ userId: entry.userId, contestId, deltaXp: award.total, reason: 'contest_finalized', breakdown: award.breakdown, seasonId: currentSeasonId, }); await tx.update(users) .set({ xpTotal: sql`xp_total + ${award.total}`, xpSeason: sql`xp_season + ${award.total}`, currentRank: sql`GREATEST(current_rank, ${rankFromXp(newXpTotal).rank})`, careerHighestRank: sql`GREATEST(career_highest_rank, ${rankFromXp(newXpTotal).rank})`, }) .where(eq(users.id, entry.userId)); }); } }
