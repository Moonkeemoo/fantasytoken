Referral System — design

Multi-tier виральна механіка. Запросив друга → ти L1 його реферер. Друг запросив когось → той L2 для тебе. Дохід — % від prizes твоїх рефералів. Замикає viral acquisition + monetization з retention loop.

NOTE: Цей mockup показує структуру UI. Конкретні числа ($50, 5%/1%) — illustrative. Canonical per-currency rates у docs/REFERRAL_SYSTEM.md §5: USD soft 5%/1%, Stars 3%/0.5%, TON 2.5%/0.5%. Welcome bonus cut з $100 → $25.

Tier tree

Referrals vs Friendships

Referrals (NEW)

  • Asymmetric
  • Immutable after first set
  • One per user (0 or 1 referrer)
  • Storage: users.referrer_user_id
  • Purpose: attribution + commission

Friendships (existing)

  • Symmetric
  • Many-to-many
  • Storage: friendships table
  • Purpose: friends leaderboard

Lifecycle

1. Share link

Лінк https://t.me/yourbot/app?startapp=ref_<tg_id>

2. Friend opens

TG SDK дає startParam=ref_X, frontend POST /friends/referral

3. Backend attribution

Set referrer_user_id якщо новий юзер (60s window + 0 entries). Immutable після цього.

4. First contest → unlock bonuses

Referee + Recruiter bonuses unlock через CurrencyService.transact()

5. Friend wins → commission

У contests.finalize: walk up chain max 2 hops, pay commissions per currency

See REFERRAL_SYSTEM.md §5 для canonical per-currency rates. Економіка моделюється у docs/mockups/referral-economics.html.
