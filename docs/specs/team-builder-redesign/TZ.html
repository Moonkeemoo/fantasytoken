03 · Allocation Sheet (M1) — детально

Модалка управління токеном

Найважливіший новий примітив. Викликається з трьох місць.

Тригери

  • Tap по token row у списку Browse tokens (Draft) — додати або підправити alloc.
  • Tap по слоту lineup (Draft) — підправити alloc вже доданого, або відкрити picker якщо слот пустий.
  • Tap по token row у Live (під час контесту) — read-only режим з PnL замість input. v1.1

Поведінка

АспектСпецифікація
Анімація відкриття Slide-up з низу екрана, cubic-bezier(.2, .8, .25, 1), 220ms. Backdrop fade-in 180ms, opacity 0 → 0.42.
Висота max-height: 62% від screen. Контент скролиться всередині, якщо overflow.
Закриття Tap по backdrop · swipe-down handle · кнопка Cancel · кнопка Add/Update (apply + close).
Default $-amount Якщо edit — existing.alloc. Якщо новий і lineup порожній → 20%. Якщо є вже інші — min(20%, remaining).
Cap behavior Slider max = 100 - sum(other.alloc). Cap-маркер на rail у позиції cap. Не дозволяти ввід > cap — обрізаємо при onChange.
$ ↔ % Two-way bind через alloc%: dollars = round(tier × alloc / 100). Inверсія при ввімкненні $: alloc = round(dollars / tier × 100).
Quick chips 10 / 25 / 50 / max(remaining). Приховувати числові chips що > cap (max — завжди).
Contest fit Bull mode + d24>0 → ✓ rising — fits. Bull mode + d24<0 → ✗ falling — fights. Bear — інверсія.
Confirm CTA Колір по mode (bull green / bear red). Текст: Add · $20,000 або Update · $20,000. Disabled при alloc = 0 або lineup full && !isEdit.
Remove Видима тільки якщо isEdit. Не disabled. Викликає onConfirm({ remove: true }).

Edge cases

  • Lineup повний (5/5) і користувач відкриває новий токен — confirm = Lineup full, disabled. Підказка: Remove a token first or tap a slot to swap.
  • Remaining = 0 (інші alloc дають 100%) — тільки edit існуючого; новий додати неможливо. Slider max = 0, всі chips disabled.
  • Користувач набирає > budget у $-input — auto-clamp до remaining. Без помилок, без shake — просто значення обрізається.
  • Token ціна змінилась поки sheet відкритий — оновлюємо price та d24 live, але alloc% (вибір користувача) не чіпаємо.
  • Backdrop tap із незбереженими змінами — закриваємо без confirm. Зміни втрачаються. (Альтернатива: confirm dialog — не робимо у v1, тримаємо легкість.)

Доступність

  • role="dialog" + aria-modal="true" + aria-labelledby на token-symbol.
  • Focus trap всередині sheet. Перший focus — на $-input.
  • Esc — закрити (як backdrop tap).
  • Slider — keyboard nav (←→ ±1%, ↑↓ ±5%).
04 · Стан та дані

Reducer, контракти, виклики

Lineup reducer — нові дії

ActionPayloadЕфект
ADD_PICK{ sym, alloc }Додає токен. Кидає, якщо length === 5 або сума > 100.
UPDATE_ALLOC{ sym, alloc }Змінює alloc існуючого. Кидає, якщо new sum > 100.
REMOVE_PICK{ sym }Видаляє токен.
RESET—Очищає lineup (для preset apply).
APPLY_PRESET{ picks: [{sym, alloc}] }Замінює lineup цілком. Сума має дорівнювати 100.

Selectors (з useDraft)

  • totalAlloc — сума alloc%, 0–100.
  • totalDollars — tier × totalAlloc / 100.
  • remainingPct — 100 - totalAlloc, для cap у sheet.
  • isValid — length === 5 && totalAlloc === 100.
  • cta — текст + state для CTA: { label, mode: 'idle' | 'pick' | 'alloc' | 'over' | 'ready' }.

API контракти

Існуюче useSubmitEntry приймає вже валідовану lineup. Очікуємо payload:

POST /entries
{
  contestId: string,
  picks: [
    { sym: 'PEPE', allocPct: 30 },  // % від tier, не $
    { sym: 'SOL',  allocPct: 25 },
    ...
  ]
}

Сервер сам обчислює $-equivalent на момент kickoff. Клієнт показує $-precomputed для UX.

05 · Live screen

Що змінюється під час контесту

Split hero

Дві однакові картки: rank та PnL. Не одна головна метрика — обидві. Гравцю однаково важливо знати, де він у таблиці і скільки заробляє.

  • Rank card: #8 (32px) + of 347 (11px) + delta ↑ +12 / 1h у pill (зелений / червоний / сірий).
  • PnL card: +$340 (24px JetBrains Mono) + +0.34% + prize estimate prize est. $61 (тільки якщо в призовій зоні).

Moment banner

Тонкий контекст-aware banner між live-banner і hero. Умови (один за раз, top priority виграє):

  • rank ≤ 10 → 🏆 You broke into top 10! Keep the lead. (gold gradient)
  • rankDelta ≥ +10 за 1h → 🚀 Climbing fast — N ranks in an hour
  • rankDelta ≤ -20 за 1h → ⚠️ Dropping fast — review your team
  • Інакше — banner не показується.

Per-token PnL row

Лівий border 3px: helping = bull-зелений, hurting = bear-червоний. "Helping" розраховується з врахуванням mode: у bear контесті падіння токена — це helping.

Local leaderboard

Top-3 → divider ↕ skip · ranks 4–N-2 → 4 рядки навколо мене (мій + 1 вище + 2 нижче).
